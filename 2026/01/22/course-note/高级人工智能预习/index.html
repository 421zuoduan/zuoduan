命题逻辑的形式可推演性: AAA 在命题逻辑中由 SigmaSigmaSigma 形式可推演, 记作 Σ⊢A\Sigma\vdash AΣ⊢A, 当且仅当 Σ⊢A\Sigma\vdash AΣ⊢A 由(有限次使用)命题逻辑的形式推演规则生成

  • 可满足: 一个句子某种真值指派中为真, 则为可满足的
  • 可靠性: 从知识库推导出的所有结论都能被知识库蕴含, 若 KB⊢αKB \vdash \alphaKB⊢α，则 KB⊨αKB \models \alphaKB⊨α
  • 完备性: 形式推演体系能推导出所有被知识库蕴含的结论, 若 KB⊨αKB \models \alphaKB⊨α，则 KB⊢αKB \vdash \alphaKB⊢α

可靠性是说形式推演出的句子一定是正确的，就是可被逻辑蕴含的；完备性是说任意能被正确的蕴含的句子，都可以被这种算法形式推出。一个推演系统是否具有完备性和可靠性是判断一个推演体系能否使用的重要标准。如果把知识库比喻一群涉案人员和一些有关犯罪嫌疑人的线索的话，我们要找的结论比喻成藏在涉案人员中真正的罪犯，那么推演体系就是找罪犯的流程。可靠性代表在推理过程中不会错误地得出一个无辜的人是罪犯的结论，而完备性代表不会漏下一个坏人。

归结原理的推演过程

  1. 知识库 KB 和待判断的句子 α\alphaα 都转换为合取范式
  2. 使用归结规则来判断 α\alphaα 能否通过 KB 推出
    归结规则: 两个已知的子句推出一个新的子句
    1. 找到两个子句中的互补文字
    2. 把这堆互补的文字抵消掉
    3. 两个子句里剩下的部分用析取连接, 得到新子句

验证句子 α\alphaα 能否被知识库 KB 推理出来

  1. 构建合取范式: 将 KB 的所有子句与 ≠α\neq \alpha​=α 合取, 构建初始合取范式
  2. 对合取范式中的子句使用归结规则, 生成新子句, 将原子句和新子句合并成新的子句集合
  3. 判断终止条件: 如果某次归结中生成空子句, 则 KB⊢αKB\vdash \alphaKB⊢α, 否则不成立

归结原理为什么能抵消: 互补文字的矛盾特性 + 子句的析取逻辑

搜索策略评价

  • 完备性：只要问题有解，算法能不能一定找到？（比如找钥匙，能不能保证找到，不管钥匙藏在哪）
  • 最优性：找到的解是不是 “最好的”？（比如找最短路径，能不能找到距离最短 / 代价最小的）
  • 时间复杂度：找解要花多久？（用 “搜索的节点数” 衡量，节点越多越慢）
  • 空间复杂度：找解要占多少内存？（用 “同时记住的节点数” 衡量，记的节点越少越省内存）

启发式搜索-A*算法

在初始状态到目标状态的所有可能路径中，高效找到代价最小的最优路径. A*算法涉及3个函数，其中，最终决定下一步走哪里的是 f(n)f(n)f(n)

  • g(n)g(n)g(n)：从初始节点到当前节点 nnn 的实际最小代价（固定值，路径确定则 g(n)g(n)g(n) 唯一）；
  • h(n)h(n)h(n)：从当前节点 nnn 到目标节点的估计代价（启发函数，决定 A* 效率与最优性）；
  • f(n)f(n)f(n)：从初始节点经 nnn 到目标的总代价估计，A* 每次优先扩展 f(n)f(n)f(n) 最小的节点。

启发函数 h(n)h(n)h(n) 要满足 “一致性”: 当前节点到目标的估计代价, 不能比先到邻结点的实际代价 + 邻结点到目标的估计代价 还大.

  • 可采纳: 启发式函数不过高估计到达目标的代价, h(n)≤h∗(n)h(n)\leq h^*(n)h(n)≤h∗(n)
  • 一致性: fff 沿着任何路径都是非递减的, 一致必可采纳 h(n)≤d(n,n邻)+h(n邻)h(n)\leq d(n, n邻) + h(n邻)h(n)≤d(n,n邻)+h(n邻)

树搜索只需要可采纳就能保证找到最优路径, 因为不会丢弃任何重复节点. 为什么图搜索使用可采纳的启发式函数不一定能找到最优路径?

图搜索为了效率引入了 closed set 可访问集合, 如果一个结点被放入, 以后遇到该结点都会跳过. 出问题的场景: h(n)h(n)h(n) 可采纳但不一致, f(n)f(n)f(n) 的值在路径上忽大忽小, 可能在节点 n 时前半段 g 大而 h 被严重低估, 这使得 f 小, 如果后面发现了一条更短的路径, 但 n 已经在 closed set 里了, 就无法更新最优路径, 找不到全局最优解

对抗搜索

Minmax 搜索 (极大极小值搜索) 通常是站在先手的视角来定义的, 所以先手要 max, 后手要 min. 从下往上回溯是因为要预测未来:

逻辑链条：

  1. 轮到 先手 (MAX) 做决定时，他必须问：“如果我走了这一步，对手会怎么做？”
  2. 他知道 对手 (MIN) 肯定会选对先手最不利（分最低）的那一步 。
  3. 所以，先手在评估自己当前的每一个选择时，其实是在比较：“如果我选 A，对手会把我逼到 3 分；如果我选 B，对手会把我逼到 2 分。”
  4. 既然 3 > 2，先手会选择 A。这就是为什么父节点的值来源于子节点的计算结果

Alpha-Beta 剪枝

核心思想: 已经知道某条路肯定不如现在的选择, 那就没必要去走这条路. 维护 α\alphaα 和 β\betaβ, 提前停止对某些分支的计算

  • α\alphaα: 先手能找到的最优下界, max α\alphaα
  • β\betaβ: 后手能找到的最优上界, min β\betaβ
  • 搜索在 [α,β][\alpha, \beta][α,β] 区间内进行, 区间外停止搜索
  1. 后手发现一个走法能使 v≤αv\leq \alphav≤α, 即比先手能找到的最好结果还要差, 则立刻停止搜索其他子节点, 直接返回. 原因: 既有算法后手最少能让先手得 3 分, 先手已知后手的方法, 所以会选择另一条路得 5 分, 后手给了新的方法会让先手只能得 2
  2. 先手发现一个走法能使 v≥αv\geq \alphav≥α, 即比后手能找到的最差结果还要好, 则立刻停止搜索其他子节点, 直接返回

作用: 在计算机 memory 一定的情况下，这个算法可以使极小极大搜索算法的深度翻倍。复杂度不变还是指数级的。

回归树

是一个分类模型, 输出连续值. 本质是将复杂的连续预测问题拆成 “层层判断 -> 分组 -> 组内平均值作为预测结构” 的过程

分类树用信息增益选特征, 回归树用均方误差的减少量选特征, MSE 越小, 说明小组内的放假越接近, 预测越准

  1. 先算数据集的初始 MSE. 如房价的平均值, 再算 (x−x‾)2(x-\overline{x})^2(x−x)2
  2. 计算每个特征的 MSE, 选最大的特征最为根结点
    比如面积分为 <50, 50-100, >100, 每组内计算 MSE, 然后根据每组的样本数量加权求和, 得到该特征的平均 MSE, 计算与整个数据集的 MSE 减少量
  3. 确定根结点
  4. 对每个分支递归构建子节点, 具体特征的那类平均值作为结果

GBRT 梯度提升回归树

  • 基本假设：多个回归树预测值相加，逐树修正前序模型残差。
  • 优化目标：最小化训练误差 + 模型复杂度惩罚。
  • 训练方法：迭代训练回归树，每棵树拟合前序模型的残差。
  • 预测方法：加总所有树的预测结果。

本质是 回归树 + Boosting + 梯度下降. 第二棵树预测原始目标值与第一棵树结果的误差, 第三棵树预测前两棵树总和剩下的误差

假设初始值是平均值, 第一棵树预测的是 T1(x)≈y−平均值T_1(x) \approx y - \text{平均值}T1​(x)≈y−平均值, 第二棵树预测的是 T2(x)≈y−(平均值+T1(x))T_2(x) \approx y - (\text{平均值} + T_1(x))T2​(x)≈y−(平均值+T1​(x))

GBRT和回归树的差异是什么？

GBRT是加性模型，它是综合了很多个小的回归树的预测结果之和。每一轮训练一个新的回归树，每一轮的训练目标实际是前面那些所有树的预测值的和与标准预测值的残差，每一轮训练的就是残差。

逻辑回归

  • 基本假设：用 sigmoid 函数将线性组合映射到 [0,1] 概率空间，本质是线性分类器。
  • 优化目标：最大化数据似然 (预测的是一个概率, 使用交叉熵损失函数)，转化为最小化 Logistic Loss，支持 L1/L2 正则化。
  • 训练方法：随机梯度下降法（SGD）更新参数。
  • 预测方法：选择概率最大的类别作为输出。

BP

  1. 梯度问题（消失 / 爆炸）：深层网络中，梯度经多层传递后急剧衰减（消失）或放大（爆炸），导致深层参数无法有效更新。
  2. 过拟合风险：当网络复杂、训练数据不足时，模型会拟合训练数据的噪声，对新数据的泛化能力差。
  3. 训练效率低：传统随机梯度下降（SGD）收敛慢；大批次训练占用内存高，小批次训练则会出现梯度波动。
  4. 调参复杂度高：隐藏层数量、神经元个数、学习率等超参数缺乏统一标准，依赖人工经验调试。
  5. 可解释性差：属于 “黑箱” 模型，难以解释各参数的实际意义与决策逻辑。

梯度消失的原因: Sigmoid 导数的最大值为 0.25, 多数情况下小于1, 网络层数较多时, 深层参数的梯度会指数级衰减, 梯度趋近于 0, 更新参数时会减去学习率×梯度, 导致深层参数无法被有效更新

  1. 所用 RNN 的网络结构
    采用RNN 编码器 - 解码器架构，流程如下：
  • 编码器：由 RNN 组成，输入法语词序列（如 “Je mange une pomme”），逐词处理并更新隐状态，最终输出包含法语语义的 “上下文向量”；
  • 解码器：由 RNN 组成，以编码器的上下文向量为初始状态，接收已生成的英语词（训练时用目标序列前一词、翻译时用自身输出词），输出下一个英语词的概率。
  1. RNN 的训练过程（以法译英为例）
    1. 准备法英平行语料，将句子拆分为词序列；
    2. 编码器逐词输入法语序列，得到上下文向量；
    3. 解码器采用强制教学策略：输入英语序列的前一个词，结合上下文向量输出当前词的概率；
    4. 计算预测词与真实词的误差，反向调整编码器、解码器的参数，重复训练至误差足够小。
  2. 翻译过程（以法译英为例）
    1. 编码器输入待翻译的法语句子（如 “Je bois du café”），生成上下文向量；
    2. 解码器从句首符开始，结合上下文向量生成第一个英语词；
    3. 后续每一步，将已生成的英语词输入解码器，继续生成下一个词，直到生成句尾符 <；
    4. 去除和 <，得到最终翻译结果（如 “I drink coffee”）。

GAN

  1. 初始化参数: 随机初始化 G 和 D
  2. 交替训练
    1. 训练判别器 D: 固定 G, 将真实数据和 G 生成的假设局输入 D ,计算 D 的分类误差 (如交叉熵), 通过梯度下降更新 D 的参数, 提升区分真假的能力

DBM 的学习

    1. 将相邻两层视为独立的受限玻尔兹曼机 (RBM)
    2. 用 RBM 的对比散度 (CD) 算法逐层训练, 得到初始参数
  2. 微调 (全局优化)

    2. 基于近似后的状态, 用对比散度或随机最大似然等算法, 调整所有层的链接参数, 优化模型对数似然, 实现全局参数优化

蚁群算法

模拟蚂蚁觅食的协作行为：蚂蚁走的路径会留下 “信息素”，路径越短、走的蚂蚁越多，信息素越浓；后续蚂蚁会优先选信息素浓的路径，最终群体通过信息素的动态积累与挥发，找到最优（最短）路径。

  1. 初始化: 放一群蚂蚁, 给所有路径初始信息素
  2. 蚂蚁寻路: 每只蚂蚁根据 “信息素浓度+路径长度” 选下一个节点, 走完全程
  3. 更新信息素: 找到的最优路径多加些信息素, 所有路径的信息素会自然挥发一部分
  4. 终止输出: 重复迭代, 直到找到最优路径

适用于离散空间的组合优化问题

  1. 旅行商问题 TSP, 车辆路径规划 VRP 等路径优化问题
  2. 车间调度, 任务分配等组合调度问题
  3. 网络路由, 图结构中的路径搜索问题

可以处理具有图结构的问题, 但收敛速度较慢, 易陷入局部最优

粒子群算法

模拟鸟群 / 鱼群的群体运动：每个 “粒子” 代表一个候选解，它会记住自己找到的最好位置（个体最优）和整个群体找到的最好位置（全局最优），然后朝着这两个位置调整飞行方向和速度，逐步逼近最优解。

  1. 初始化: 生成一群例子, 随机给它们初始位置和速度
  2. 评估好坏: 计算每个粒子当前位置的优劣
  3. 记录最优: 每个粒子记自己的最好位置, 整个群体记全局最好位置
  4. 调整飞行: 粒子根据 “惯性+个体经验+群体经验” 调整速度和位置, 向最优位置靠近
  5. 终止输出: 重复迭代, 直至找到稳定的最优解

适用于连续空间的优化问题

  1. 函数优化, 如高维, 非线性函数的极值搜索
  2. 神经网络参数优化, 支持向量机参数调优
  3. 工程优化 (如结构设计, 参数配置), 多目标优化问题

收敛速度快, 实现简单, 但易陷入局部最优, 对参数如惯性权重较敏感

贝尔曼方程: 定义了状态估值函数的依赖关系, 所有状态的估值依据贝尔曼方程形成了一个具有 n 个方程和 n 个变量的线性方程组, 求解方程组可得到该策略下每个状态的估值

状态估值: 它是在当前策略下，从某个状态（比如某个格子）出发，未来能获得的累计奖励的期望值

格子问题中, 上下左右的奖励一致时, 有

$V(s) = \sum \pi(a|s) \times [\text{即时奖励} + \text{未来价值}]
