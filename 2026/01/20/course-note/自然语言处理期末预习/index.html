生成式模式的编解码模型, 评价指标

序列生成模型

模型结构:

  • 纯生成式模型
    • 基本 RNN 架构
    • RNN + Attention
    • Transformer
  • 选择式模型
    指针网: 输出是输入的子集, 凸包问题, 可以解决 OOV 问题
  • 选择-生成式模型
    输出从输入中选择, 输出与输入词表相同, 无法处理 OOV 问题

训练策略:

  • 自编码(AE, AutoEncoder): 输入与学习目标相同, 隐含层即编码特征.普通自编码器存在潜变量空间不连续, 无法通过随机采样生成新数据, 容易过拟合等问题, 所以提出 VAE 变分自编码器模型, 引入概率分布来约束潜变量空间, Encoder 映射到一个正态分布上, 冲参数化得到潜变量
  • 对抗生成网络
  • 扩散生成模型
    优势:
    1. 高质量数据生成
    2. 训练稳定

序列生成评价指标

  1. 正确率 precision: 在模型预测为正类的样本中，有多少是真正的正类？（即：预测的“准”不准）
  2. 召回率 recall retio: 在实际为正类的样本中，模型找出了多少？（即：找的“全”不全）
  3. BLEU: 生成序列与参考序列间的 N-Gram 重合度, 加上生成重复惩罚, 长度过短惩罚. [0, 1], 越大越好, 只计算精度, 不关心召回率
  4. ROUGE: 与出现次数有关

序列生成模型问题:

  1. 曝光偏差问题: 训练时前缀是真实数据 (预测第t个词的时候, 前面的词都是真实标签, 这就是 Teacher Forcing, 否则训练的模型会越走越远, 有误差累计), 测试是模型生成的前缀, 前缀有错误的时候后续序列也会偏离真实分布
    解决方法: 训练中混合使用真实数据和模型生成数据, 随训练次数的增加减小使用真实数据的量
  2. 训练-评价目标不一致: 训练使用最大似然估计, 测试使用 BLEU, ROUGE
    解决方法: 采用强化学习策略进行模型训练, 将最回归序列生成看成一种马尔科夫决策过程

BPE (Byte Pair Encoding)

一种子词分词算法, 处理 OOV 词表问题. 常用词保持完整, 生僻词拆分成常见的词根/子词, 比如 “unhappiness” 分成 “un”, “happi”, “ness”

  1. 准备语料与初始化: 统计语料库中相关词出现的频次. “low” 5, “lower” 2, “newest” 6, “widest” 3
  2. 统计相邻字符对的频率. 比如 “e” 和 “s” 在 “newest” 和 “widest” 都出现了, 频次 6+3=9
  3. 合并最高频的字符对. “e” 和 “s” 合并为最高频的字符对
  4. 继续迭代. “es” 和 “t” 挨在一起, 合并为 “est”, 频次 6+3=9. 直至得到 “est”
  5. 处理其他高频对. 直至达到预设的词表大小, 然后用新的词表重新训练
  6. 分词阶段 (inference). 遇到新词 “lowest”, 拆分字符, 按训练时学到的合并顺序, 优先级从高到低进行合并: "l o w es t " -> "l o w est " -> “l o w est” -> "lo w est " -> low est

为什么加 “”: 为了区分单词结尾与单词中间的字符序列

循环神经网络 RNN

单一 DNN, CNN 无法处理时序相关序列问题, 循环神经网络在时序上分解成一系列单元, 单元的神经网络能在时序上在展开, 且能将上一时刻的结果传递给下一时刻

其中，XtX_tXt​ 是当前时刻输入，h(t−1)h(t-1)h(t−1) 是上一时刻的隐藏状态，WiW_iWi​ 和 WhW_hWh​ 是权重矩阵. 输出 Yt=softmax(Woh(t))Y_t = \text{softmax}(W_{o}h(t))Yt​=softmax(Wo​h(t))

LSTM

为解决长序列训练中的梯度消失和长期依赖问题

LSTM 的核心设计思想是引入一个细胞状态 (Cell State, CtC_tCt​)，它贯穿所有时间步，像一条传送带一样，信息可以在上面轻松流动并保持不变。通过三个精心设计的“门”结构来控制信息的添加与遗忘。

(1) 核心组件：三个门, LSTM 通过 Sigmoid 神经网络层（输出0到1之间的数值）来控制信息流：

  • 遗忘门 (Forget Gate, ftf_tft​)：作用：决定从上一时刻的细胞状态 Ct−1C_{t-1}Ct−1​ 中丢弃哪些信息。机制：读取上一时刻的隐藏状态 ht−1h_{t-1}ht−1​ 和当前输入 xtx_txt​，输出一个 0 到 1 之间的向量。1 表示“完全保留”，0 表示“完全丢弃”。
  • 输入门 (Input Gate, iti_tit​)：作用：决定在当前细胞状态中保存哪些新信息。机制：包含两部分：Sigmoid 层决定更新哪些值。Tanh 层创建一个新的候选值向量 C~t\tilde{C}_tC~t​。
  • 输出门 (Output Gate, oto_tot​)：作用：决定基于当前的细胞状态，输出什么样的隐藏状态 hth_tht​。机制：首先运行一个 Sigmoid 层来确定细胞状态的哪个部分将输出，接着把细胞状态通过 Tanh 进行处理（得到 -1 到 1 之间的值），并将两者相乘。

(2) 运作流程遗忘阶段：ftf_tft​ 决定旧状态 Ct−1C_{t-1}Ct−1​ 中哪些部分被遗忘。记忆阶段：iti_tit​ 与候选状态 C~t\tilde{C}_tC~t​ 结合，向细胞状态中添加新信息。更新公式：Ct=ft∗Ct−1+it∗C~tC_t = f_t * C_{t-1} + i_t * \tilde{C}_tCt​=ft​∗Ct−1​+it​∗C~t​输出阶段：计算最终的隐藏层输出 ht=ot∗tanh⁡(Ct)h_t = o_t * \tanh(C_t)ht​=ot​∗tanh(Ct​)。

损失函数本质是交叉熵损失函数, 每个时间步预测的词应与标签的交叉熵求和

优点:

  1. RNNLM 可以保留每个词的全部历史信息, 不需简化为n-gram
  2. 引入词向量作为输入后不需要数据平滑 (训练时没有"吃橘子"但有"吃苹果", 也能输出"吃橘子")

注意力机制

注意力机制就是加权求和机制

  • Hard Attention: 一个为 1, 其他为 0
  • Soft Attention: 一个概率分布

优势:

  1. 找到输入相关信息, 提高输出质量
  2. 不需要监督信息, 对先验认知少的问题有效
  3. 解决长距离依赖问题

Q 序列与 K 序列融合编码, 输出序列长度是 Q 序列的长度

多头注意力机制: 能从不同的表示子空间中学习相关信息

Transformer

模型特点

  1. 全部采用 Attention 机制
  2. 高度并行, 训练速度快
  3. 能捕捉长程依赖信息, 有较高的建模能力 (训练并行, 但是测试时编码端并行, 解码端串行)

self−attn(Q,K,V)=softmax(QKTdkV)\mathrm{self-attn}(Q, K, V) = \mathrm{softmax}\big(\frac{QK^T}{\sqrt{d_k}}V \big) self−attn(Q,K,V)=softmax(dk​​QKT​V)

  • Attention:
    • 输入线性变换: 将QKV映射到不同的子空间
    • dk\sqrt{d_k}dk​​: 避免内积过大时, softmax 产生的结果非 0 即 1
    • 层归一化: 对数据进行重新放缩, 提升模型训练稳定性
    • MHSA: 不同头为 Attention 提供了不同的表示子空间, 不同表示关注不同位置 (Encoder-Decoder 的多头注意力机制作用: 链接源序列和目标序列, 根据需要从源序列中提取上下文信息; 动态关注不同部分)
    • mask:
    1. padding mask: 处理非定长序列问题, 对加长的数据位置的值加上负无穷, softmax 后权重就会接近 0, 从而不会对 attention 后的向量起作用
    2. sequence mask: 防止标签泄露
  • FFN: 从其他 token 获取信息后, 融合和处理信息

Transformer的问题在于固定窗口大小和二次空间复杂度, 这阻碍了其在长文档理解中的应用

BERT

堆叠多个双向 Transformer Encoder, 不使用 Decoder

预训练策略:

  1. 看不到自己: MLM 随机 mask 部分输入 token, 预测被屏蔽的 token
    问题:
    1. [MASK] 在微调时没出现, 预训练和微调间不匹配: 15% token进行遮盖, 其中 80% 换成 [MASK], 10% 替换另一个语料中的词, 10%不处理. 处理后模型不知道被要求预测哪些单词或哪些单词被随机单词替换, 所以被迫保持每个输入 token 的上下文表示
    2. 只有 15% 数据被预测, 需要更多的预训练来收敛
  2. 建模句间关系: 使用二元分类任务预训练句子关系模型

训练方法

  1. 屏蔽 token: 随机掩码的训练过程漫无目的且效率低效, 指从以小组输入 token 中学习, 可以在反向传播中根据重要性选择性地屏蔽一部分 token 来加速模型训练
  2. 预热策略: 训练时线性增加学习率, 不同层适应地使用不同学习率
  3. 共享相似的自注意力模式: 训练浅层模型, 复制浅层模型以构建生成模型

大模型指令学习, 上下文学习, 思维链等概念, 各类训练方法

提示学习设计原则

  1. 预训练模型选择: PLM模型结构多样，同时预训练任务也各有特色，选择与特定任务匹配的PLM模型十分必要
  2. Prompt Engineering: 针对不同任务，选择对应合适的Prompt输入至关重要
  3. Answer Engineering: 主要涉及模型输出的粒度形式（token,span,sentence,etc.）及模型输出到实际标签的映射方式
  4. Multi-Prompt Learning: 相对于Single Prompt, Multiple Prompt学习能在一定程度上提升最终效果
  5. Training Strategies: 训练Prompt的方法多种多样，LM参数及Prompt参数可进行多种组合训练

上下文学习

上下文学习使用由任务描述和（或）示例所组成的自然语言文本作为提示

示例选择:

  1. 基于相关度排序: 如 K 近邻方法
  2. 基于集合多样性的方法: 针对特定任务选出具有代表性的, 信息覆盖性好的示例集合
  3. 基于大语言模型的方法: 大模型作为评分器对候选样本进行评估

示例顺序:

  1. 基于随机选择
  2. 基于语义相似度: 与测试样本语义相似的示例放在更靠近测试样本的位置

评估示例顺序质量:

  1. 基于数据集: 测试大模型基于该示例顺序的任务性能, 作为当前示例顺序的评分
  2. 基于结果不确定性: 计算基于该示例顺序大语言模型预测分布的熵值, 选择熵值较低的示例顺序作为较为有效的顺序

  • RM: 采样 prompt, 让第一步训练的 SFT 生成可能的答案, 人工对答案进行排序, 构建 prompt + 排序答案集合, 训练 RM

事件抽取概念和子任务

事件抽取: 主要研究如何从描述事件信息的文本中抽取出用户感兴趣的事件信息并以结构化的形式呈现出来

本质: 从无结构化数据中抽取结构化事件信息

步骤:

  1. 触发词检测: 抽取描述事件发生的核心动词或名词并确定其类别
  2. 确定事件论元集合: 对指定事件确定需要抽取哪些角色
  3. 抽取/识别事件论元: 抽取参与事件的实体及其角色 (如参与者, 事件, 地点)

子任务:

  1. 句子级事件抽取
    1. 触发词检测 (标注+分类)
    2. life/Be-Born 类模板: Person, Time, Place
    3. 按模板抽取角色 (标注)
  2. 文档级事件抽取
  3. 小样本/零样本事件抽取

属性级情感分析概念及建模方法

细粒度属性级情感分类是指判断评价对象及其属性的情感倾向

基于属性的情感分析 Aspect-based Sentiment Analysis（ABSA）: 更精细的情感分析，为每个实体（及其属性）识别情感, 能更好地理解主观性文本的情感分布. 属性及情感分类关注的观点为常规性观点, 即对某些目标实体的情绪/意见表达. 任务形式主要为抽取和分类

建模方法:

  1. 早期深度学习方法 (RNN/LSTM Based)
    这一阶段主要利用RNN/LSTM及其变体来处理序列数据，并引入注意力机制来增强模型对特定属性的感知能力。

  • 属性抽取 (Aspect Extraction, AE)

    • 模型框架：采用Seq2Seq结构，结合双向GRU（Bi-GRU）。
    • 核心机制：
      • 门控单元网络：编解码器采用GRU单元。
      • 位置感知注意力：在关注内容的同时考虑相对位置信息，提升抽取准确率。
      • 序列标注：通常将抽取任务视为序列标注问题（如BIO标注）。

  • 属性级情感分类 (ALSC)

    • AT-LSTM (Attention-based LSTM)：利用注意力机制捕捉句子中与给定属性（Aspect）相关的部分，将属性向量与LSTM的隐藏层输出进行交互计算注意力权重。
    • ATAE-LSTM (Attention-based LSTM with Aspect Embedding)：
      • 输入层改进：将属性向量直接拼接到输入层的词向量中，使模型在编码阶段就感知到目标属性信息。
      • 注意力机制：同样使用Attention关注句子的不同部分。

  • 面向属性的观点抽取 (AOE)

    • IO-LSTM：
      • Encoder设计：包含三个LSTM模块。向内LSTM（Inward-LSTM）将上下文信息传递给目标属性；向外LSTM（Outward-LSTM）将目标属性信息传递给上下文；IO-LSTM获取全局上下文信息。
      • Decoder：每个位置独立进行B/I/O三分类预测。
  1. 基于跨度与多任务学习的方法
    为了解决流水线方法的错误传播问题，研究者开始探索基于跨度（Span）的抽取和多任务联合学习。

  • 跨度分类流水线

    • 框架：先抽取后分类（Extract-then-Classify）。
    • 流程：首先利用基于BERT的抽取器预测属性的开始和结束位置；然后利用分类器直接使用抽取出的属性跨度特征进行情感分类。

  • 联合抽取 (SpanMlt)

    • 模型：基于跨度的多任务学习框架。
    • 机制：
      • 共享编码：使用BERT或BiLSTM作为共享编码器。
      • 枚举跨度：枚举所有可能的跨度，并生成跨度表示。
      • 评分器：包含术语评分器用于识别实体/观点，关系评分器用于配对。
  1. 联合与生成式框架
    这一阶段致力于统一输入输出形式，通过一个模型解决多个子任务，实现端到端的联合抽取与分类。

  • 基于阅读理解的联合抽取 (Dual-MRC)

    • 核心思想：将三元组抽取转化为机器阅读理解（QA）任务。
    • 架构：双BERT结构，参数共享。一个BERT负责属性抽取，检测开始/结束位置；另一个BERT负责观点抽取和情感分类。

  • 统一生成式框架

    • 模型：基于BART的Seq2Seq模型。
    • 核心思想：将所有ABSA子任务（如属性抽取、观点抽取、情感分类、配对、三元组抽取）统一为生成任务。
    • 输入输出：输入原始句子序列，输出包含指针索引和类别索引的序列。
  1. 基于大模型的方法 (LLM-based)
    利用大语言模型（LLM）的推理能力和少样本学习能力进行ABSA任务。

  • 推理增强

    • Extract-Then-Assign：利用思维链（Chain-of-Thought）提示，引导LLM先提取相关项再分配情感标签。
    • Self-Consistency：通过多样化推理路径生成多个候选四元组，利用一致性投票筛选最终结果。

  • 数据增强与鲁棒性

    • 反事实样本生成：利用LLM生成高质量的反事实样本（如修改情感词），解决虚假相关性问题。
    • 信息瓶颈：在训练中平衡事实与反事实部分，学习鲁棒的表示。

  • 检索增强

    • 框架：包含示例检索和任务推理两个阶段。
    • 机制：检索器为给定样本选择相似的候选示例（Few-shot examples），辅助语言模型进行打分和预测。

任务型对话方法设计思想及典型模型

任务型对话系统的核心目标是帮助用户完成特定领域的具体任务（如订票、查询天气等），其核心设计通常涉及对用户意图的理解、状态的追踪以及策略的执行

pipeline 式对话系统

  1. 自然语言理解 (NLU)：将用户的自然语言输入转换为计算机可理解的语义标签，包含意图识别（分类任务）和语义槽填充（序列标注任务） 。CRF & Bi-LSTM
  2. 对话状态追踪 (DST)：根据对话历史和当前输入，维护当前的对话状态（通常是一组槽值对），以处理多轮交互中的信息累积 。BERT for Joint Learning, T5 将 DST 任务建模乘生成任务
  3. 对话策略学习 (DPL)：基于当前的对话状态，决定系统下一步的动作（Action），如询问、确认或提供信息。DST和DPL合称为对话管理（DM） 。RL 将对话视为序列决策过程, PPDPP 基于 LLM 的策略规划期
  4. 自然语言生成 (NLG)：将抽象的系统动作转换为自然流畅的文本回复反馈给用户 。

端到端设计

为了解决流水线各模块独立优化导致的误差累积问题，端到端设计直接学习从输入（对话历史）到输出（系统回复）的映射，通常基于神经网络模型 。如 SOLOIST, 基于 GPT-2 微调; PPTOD, 多任务提示学习

基于大模型的设计

  • Prompt-based（基于提示）：通过构造Schema、规则和对话历史作为Prompt，直接让LLM进行状态追踪或策略规划，无需大量标注数据 。
  • Instruction Tuning（指令微调）：通过Adapter或LoRA等技术对大模型进行轻量级微调，使其适应特定领域的对话任务 。
