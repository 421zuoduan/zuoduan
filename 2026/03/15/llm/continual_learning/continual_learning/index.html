ViT + 图像分类

[CVPR2022] Learning to Prompt for Continual Learning

rehearsal-free: 不保存旧数据时, 顺序学习多个任务且不遗忘旧任务的持续学习能力

L2P: 针对持续学习中的图像分类, 用一个 prompt 代表一个任务, 用 prompt 来存储特定任务的处理能力, 根据样本从 prompt pool 动态选择 prompt, 相似任务共享 prompt, 不同任务少共享 prompt

  1. 将图像输入冻结 backbone, 得到图像编码特征, 在 ViT 里就是 [class] token
  2. 将图像编码特征与预存储的 key 进行特征匹配, 找到 top-N 的 prompt
  3. prompt 与输入 embedding 拼接, 重新输入冻结 backbone

[ECCV2022] Visual Prompt Tuning

严格来讲不是持续学习的论文. 系统性地将 visual prompt tuning 引入视觉模型微调, 本质是参数高效微调. 在 ViT 输入空间加入可学习 prompt, 训练 prompt 和线性头, 冻结整个 backbone.

有两个变体, VPT-shallow 只在第一层输入前加 prompt, VPT-deep 在每个 Transformer encoder layer 的输入前都加 prompt

[ECCV2023] DualPrompt: Complementary Prompting for Rehearsal-free Continual Learning

L2P 所有的知识都混在一个 prompt pool, 把 prompt 显式地分成了两类:

  • G-Prompt: 跨任务共享的知识
  • E-Prompt: task-specific 知识

G-Prompt 不需要检索, 所有任务共享, E-Prompt 需要检索. 实验发现前者更适合浅层, 后者更适合深层

[ICLR2026] ONE-PROMPT STRIKES BACK: SPARSE MIXTURE OF EXPERTS FOR PROMPT-BASED CONTINUAL LEARNING

Motivation: 对于多任务系统的持续学习, 要么为每个任务分配独立的提示, 这会导致参数量随任务增加而爆炸, 要么所有任务共享同一提示词. 这存在一种多提示参数爆炸与单提示灾难性遗忘之间的权衡困境

Method: MoE prompt, 提出 SMoPE. 采用 prefix tuning, 每层 attention 里都有 prefix key $P^K$ 和 prefix value $P^V$，作者把其中prompt expert。然后在每个 MSA 层里，根据当前层输入算出每个 expert 的代理分数，只选 top-K 个 experts 参与 attention

既有方法都是拿 frozen backbone 的输出与需要训练的 task key 去算 gated score, 这篇论文扩大了 attention map, score 来自所有 token, 需要训练的是 attention 里的 key

  1. 第一个任务做 dense warm-up. 不做 prompt 的 top-K 选择, 所有 prompt experts 都参与. 第一个任务用 $E/2$ 个 epoch 做 dense training, 优化交叉熵
  2. sparse training. 每个 prompt 与输入 token 都有一个 attention score, 对其取平均作为该 prompt 的重要性
  3. 高频激活惩罚, 为了避免模型总会选同一批 experts, 模型维护一个累计激活频率, 训练时对激活频率作一个高频惩罚
  4. 取惩罚后 attention score 的 top-K, 将选中的 prompt expert 作为 prefix 加在输入上, 另外相应的将 prefix key/value 输入 prompt attention
  5. $L = L{ce} + \alpha{router}L{router}+\alpha{proto}L_{proto}$, proto 是用来保旧知识的, 当换下个任务的时候, 上一个任务激活最多的 prompt expert 应该保持尽量路由一致, 输入上一个任务的数据的时候可以使用这几个 prompt expert.
  6. 分类头预测
  7. 更新 expert 的使用频率

task-adaptive-prediciton (TAP): sparse training 后, 用额外的 $L_{tap}$ 去优化分类头 $\phi$, 能缓解持续学习里分类头偏向新类的问题

[ICLR2026] FlyPrompt: Brain-Inspired Random-Expanded Routing with Temporal-Ensemble Experts for General Continual Learning

作者将 MoE 的路由与 expert 能力分开处理. 将 generative continual learning 分成了 expert routing 和 expert competence 两个问题, 前者用 REAR 做解析式路由, 后者用 TE^2 做多时间尺度的分类头集成

generative continual learning: 训练数据流分多个 session 到来, 类别会重复出现, 每个 session 的活跃类别数会变, 样本比例也会变, 测试时没有任务 ID. 在线学习是数据流式到来, 不回头看旧数据

输入：一条连续到来的图像数据流，以及一个预训练好的 ViT-B/16 backbone。不同 session 对应一个 expert/prompt，但测试时要靠路由器自己选。

输出：对当前输入样本的类别预测。中间会先选出一个 expert，再用该 expert 的 prompt 和多个头做最终分类。

REAR: 解决 MoE 的 routing 问题, 把特征映射到高维空间, 用给一个解析式回归器学这个样本属于哪个 expert, 给出了 routing 的闭式解

TE^2: 解决分类头可以因为分布漂移而效果变差的问题. 给每个 expert 配几个不同的分类头, 分类头初始化相同, 用不同的衰减率更新 expert 的不同分类头, 大多数投票

[ICLR2026 Oral] PLUG-AND-PLAY COMPOSITIONALITY FOR BOOSTING CONTINUAL LEARNING WITH FOUNDATION MODELS

Motivation: 既有方法没有显式利用跨任务共享 (不同任务里的不同类别共享一部分相同的概念, 比如两个任务都是识别动物, 那么动物腿可能是一种可以共享的抽象概念) 的概念结构, 这种信息如果作为训练信号能提升持续学习效果

  • patch: 固定 N 个
  • slot: 固定 K=10,

  • primitive: 1 个, 从所有 slot 中聚合成一个 primitive 向量

  • 第一阶段（concept learning）：先训练一个“概念提取器”

    1. 图片输入 ViT backbone 提取 patch 特征
    2. patch 特征输入 slot attention, 得到一组 slots 和对应的 attention mask
    3. 给 slot 加上位置编码, 经过 decoder 和attention mask 合成回去, 用 slot 重建原 patch 特征, 这样可以保证 slot 不是为了分类凑巧可用, 而是真的解释了图像. 这里有一个重建 loss
    4. slots 经过非线性映射, 与可学习的 primitive key 算相似度, 所有 slots 及全球和, 得到单个 primitive 表征. 如果两张图像标签相同, primitive 应该相似, 所以有一个衡量类内一致性的 primitive loss. 这一阶段总的 loss 是 $L = L{reconstruction} + \alpha L{p}$
  • 第二阶段（knowledge distillation）：再把这个概念提取器学到的关系，拿去监督原来的持续学习模型 logits
    1. 计算 batch 内 primitive 上的样本相似度矩阵和 logits 上的样本相似度矩阵
    2. 用 primitive 提供监督, 把这种关任务的关系蒸馏给 logits, 计算 KL 散度得到 alignment loss, 最终 loss 是 $L = L{ce} + \alpha L{alignment}$

slot attention: 来自论文 Object-Centric Learning with Slot Attention. 

1. 接收一组大小为 $N$ 的输入特征 (本文为 patch embedding), 初始化 $K$ 个从可学习参数的高斯分布中采样得到的 slot 向量
2. 对输入作 KV 映射, 对 slots 做 Q 映射, 做注意力机制 (得到 $N\times K$ 的注意力图, 有点像交叉注意力机制)
3. 最终训练得到的 slots 相当于对所有 patch 信息作聚合, 得到多种图像上的 pattern 和他们对应的 attention mask (mask 的作用是知道这个 slot 聚合了图上哪些 patch 的信息)

[arXiv:2602.01990v1] SAME: Stabilized Mixture-of-Experts for Multimodal Continual Instruction Tuning

路由网络的更新投影到与旧任务正交的子空间可以稳定专家选择. 这篇论文是在 LLM 领域做的工作, 只训练了 lora 和 multimodal projector

SAME (Stabilized Mixture-of-Experts)

  • 谱感知路由 (Spectral-aware Routing): 针对路由漂移. 只在 router 上做正交投影更新, 只更新任务相关的 expert

  • 曲率感知缩放 (Curvature-aware Scaling): 针对专家漂移. 损失函数的局部曲面上, 如果参数的微小变化导致 loss 剧烈增加, 说明这个地方的曲率很大 (Fisher 信息矩阵对应的值大). 作者维护一个历史输入的协方差矩阵来近似曲率, 利用黎曼公式的缩放机制使旧任务的 expert 更新缓慢

    曲率指的是损失函数在参数空间上的二阶导数矩阵 (黑塞矩阵), 深度学习中精确计算黑塞矩阵不太现实, 学术界经常使用 Fisher 信息矩阵 (FIM) 来近似, 核心定义是一阶梯度平方的期望 $F = \mathbb{E}[(\nabla_W \mathcal{L})(\nabla_W \mathcal{L})^T]$

    线性层的一阶梯度可以表示为 $\nabla_W \mathcal{L} = (\nabla_z \mathcal{L}) x^T$, $z=Wx$

    矩阵展开, $F \approx \mathbb{E}[(\delta x^T)(\delta x^T)^T] = \mathbb{E}[\delta \delta^T x x^T]$, 一般认为误差项 $$ 与输入特征 $x$ 在统计上是弱相关或收敛点附近的方差趋于常数, 所以是正比关系 $F \propto \mathbb{E}[x x^T] = \Sigma_{old}$

    综上, 历史输入的协方差矩阵可以近似曲率

    普通梯度下降公式 $\Delta W = - \eta \cdot g_{new}$

    基于黎曼梯度的梯度下降公式 $\Delta W = - \eta \cdot (\text{曲率矩阵的逆}) \cdot g{new}$, 也即 $\Delta W = - \eta \cdot (I + \lambda \Sigma{old})^{-1} \cdot g_{new}$, 所以曲率大的地方梯度更新速度慢

[ICLR2026 Workshop LLA] Self-Distillation Enables Continual Learning

LLA: lifelong agents

自蒸馏 + 持续学习. 主流的 demostration-based 方法是 off-policy 的, 训练和测试的状态分布不一致, 这篇论文利用教师学生为相同模型实现了 on-policy, 提出 SDFT（Self-Distillation Fine-Tuning）. 这篇论文完成 skill-learning (给新任务数据和 demos, 学会新任务且不失去原有通用能力), knowledge acquisition 两类学习问题 (把预训练的新知识注入模型, 也能在简介提问上泛化)

off-policy: 训练用的是别的策略产生的数据，因此可能和当前策略实际运行时遇到的分布不一致. SFT 像 off-policy

on-policy: 训练用的是当前策略自己产生的数据，因此更贴近当前策略实际运行时遇到的分布. 本文提出的是 on-policy

Motivation: 主流的 demonstration-based 学习方式 SFT 本质上是 off-policy 的，训练分布和测试时模型自己会走到的状态分布不一致，所以很容易产生遗忘和泛化差

Method: 教师能看到 question+demo, 学生只能看到 question, 在学生的生成轨迹上做蒸馏

  1. 当前 student 只看，自己生成回答轨迹；
  2. teacher 看问题 + demonstration，在这条 student 轨迹上给出分布；
  3. 用这个蒸馏信号更新 student；
  4. 然后用 EMA 把 teacher 参数朝着 student 轻微移动一点

EMA (Exponential Moving Average): 指数滑动平均

$\thetat$ 是 student 参数, $\phi_t$ 是 上一时刻的 teacher 参数, $\alpha$ 极小. 单次更新参数有 $\phi{t+1}​=\alpha\theta_t​+(1−\alpha)\phi_t​$

$\phi​{t+1} = \alpha\theta_t + (1-\alpha)\alpha\theta{t-1}+(1-\alpha)^2\alpha\theta_{t-2}+\cdots$, 这样就满足 “指数” 滑动平均了

这篇论文里, student 看一份 batch 并改参数, teacher 用 student 参数改参数. 引入 EMA 的作用是让 teacher 能跟上 student 的进步, 但不要跟得太快. 原文 “track the student’s progress while smoothing high-variance updates”

作者实验设置了两组, frozen teacher 训练稳定但效果持续更差, 因为 teacher 不能反应 student 在训练中已经学到的东西; current student as teacher 存在学生模型单步更新有瞬时噪声的问题, 如果 current student as teacher 会导致 teacher 不平滑不稳定, 而且波动会通过 on-policy 闭环放大, EMA teacher 可以在跟踪 student 的同时平滑高方差更新
