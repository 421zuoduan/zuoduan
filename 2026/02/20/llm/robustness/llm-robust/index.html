Certified Robustness 可证明鲁棒性可分为两种学派: 概率统计派 (基于随机平滑 Randomized Smoothing) 与确定性推导派 (Deterministic Certification). 可认证鲁棒在数学上保证了只要扰动在特定半径内，模型的预测结果不会发生改变

随机平滑

[NIPS2019 Short Talk(Spotlight)] Certified Adversarial Robustness via Randomized Smoothing

TLDR

构建过程：

  1. 基础分类器：假设我们有一个任意的、可能不鲁棒的神经网络 f(x)f(x)f(x), 这个分类器在训练时要有一定的加噪数据, 已经有一定的对噪声数据的识别能力
  2. 添加高斯噪声：对于输入 xxx，我们向其添加服从均值为 0、标准差为 σ\sigmaσ 的各向同性高斯噪声 ϵ∼N(0,σ2I)\epsilon \sim \mathcal{N}(0, \sigma^2 I)ϵ∼N(0,σ2I)。
  3. 多数表决：平滑分类器 g(x)g(x)g(x) 的输出，是基础分类器 fff 在添加噪声后，最有可能输出的类别。

具体分析:

在输入空间 Rd\mathbb{R}^dRd 中，当分类器 fff 的预测基于高斯平滑分布 N(x,σ2I)\mathcal{N}(x, \sigma^2I)N(x,σ2I) 时，如果平滑分类器 g(x)g(x)g(x) 以置信下界 pA‾\underline{p_A}pA​​ 预测出类别 cAc_AcA​，那么在 l2l_2l2​ 半径为 RRR 的球体内，g(x+δ)g(x+\delta)g(x+δ) 的预测类别绝对不会发生改变（即依然为 cAc_AcA​）

R=σ⋅Φ−1(pA‾)R = \sigma \cdot \Phi^{-1}(\underline{p_A}) R=σ⋅Φ−1(pA​​)

其中 σ\sigmaσ 是平滑标准差, 也即噪声的标准差; pAp_ApA​ 是多数类的置信下界, 即在置信度 1−α1-\alpha1−α 下的最差概率 pAp_ApA​; Φ−1\Phi^{-1}Φ−1 是标准正态分布的累积分布函数的逆, Φ(z)=∫−∞z12πe−t2/2dt\Phi(z) = \int_{-\infty}^{z} \frac{1}{\sqrt{2\pi}} e^{-t^2/2} dtΦ(z)=∫−∞z​2π​1​e−t2/2dt.

对于随机平滑, 假设样本是高维空间内的一个点, 后门攻击时对样本加噪使加噪样本相比原始样本更靠近高维的有害空间, 此时再加噪, 相当于是有害加噪样本为中心的一个球, 此时这个也有很大一部分是在有害空间的, 为什么能做到防御攻击呢?

答: 因为噪声尺度并不大, 加噪样本比原始样本仅移动一小部分, 高维空间内判定有害可能性更大的空间想象成一个锥形, 大多数投票后判定仍然是安全

Trade-off

σ\sigmaσ 越大, 噪声越大, 但是干净图像的准确率会降低, 所以需要做 Trade-off, 这就引入了 ACR (Average Certified Radius 平均认证半径)

ACR=1∣Dtest∣∑(x,y)∈DtestR(x,y)⋅I(预测正确)ACR = \frac{1}{|D_{test}|} \sum_{(x,y) \in D_{test}} R(x, y) \cdot \mathbb{I}(\text{预测正确}) ACR=∣Dtest​∣1​(x,y)∈Dtest​∑​R(x,y)⋅I(预测正确)

遍历测试集 DtestD_{test}Dtest​ 中的所有图片; 如果模型把这只猫认错了，那这只猫提供的半径贡献直接记为 000; 如果认对了，就把算出来的认证半径 RRR 累加起来; 最后除以测试集总样本数，求平均

优缺点

优势 (Pros)：

  1. 模型不可知 (Model-Agnostic)：不需要知道 fff 的内部结构，把 fff 当作黑盒即可。
  2. 可扩展性极强 (Scalability)：完美适配 ResNet 等大型网络和 ImageNet 等大型数据集的。

局限性 (Cons)：

  1. 推理成本高昂：为了对一张图片进行分类和认证，可能需要进行 10410^4104 到 10510^5105 次前向传播（因为要蒙特卡洛采样）。
  2. 概率性认证：认证结果不是 100% 确定的，而是以 1−α1 - \alpha1−α 的置信度成立（尽管 α\alphaα 可以设置得很小）。
  3. 准确率权衡 (Accuracy Trade-off)：加入较大的噪声 σ\sigmaσ 虽然能提升鲁棒半径，但也破坏了图像原本的特征，会导致模型在干净样本上的标准准确率下降。

公式推导如下:

设基础分类器为 (f:\mathbb{R}^d\to\mathcal{Y})，噪声为 (\varepsilon\sim \mathcal{N}(0,\sigma^2 I))，平滑分类器定义为
[
g(x)=\arg\max_{c\in\mathcal{Y}}\mathbb{P}\big(f(x+\varepsilon)=c\big).
]
记当前输入 (x) 处的预测类别为 (A)，即 (A=g(x))。进一步记
(p_A=\mathbb{P}(f(x+\varepsilon)=A))，
(p_B=\max_{c\neq A}\mathbb{P}(f(x+\varepsilon)=c))。
对任意扰动 (\delta\in\mathbb{R}^d)，目标是推出一个半径条件，使得当 (|\delta|_2) 足够小时仍有 (g(x+\delta)=A)。

对任意类别 (c)，定义集合
[
S_c:={z\in\mathbb{R}^d:f(z)=c}.
]
于是
[
\mathbb{P}(f(x+\varepsilon)=c)=\mathbb{P}(x+\varepsilon\in S_c).
]

令
[
X\sim \mathcal{N}(x,\sigma^2 I),\qquad Y\sim \mathcal{N}(x+\delta,\sigma^2 I).
]
则
[
\mathbb{P}(X\in S_A)=p_A,\qquad \mathbb{P}(X\in S_c)\le p_B\quad(c\neq A).
]

由高斯测度在平移下的最优界，若 (\mathbb{P}(X\in S)\ge p)，则
[
\mathbb{P}(Y\in S)\ge \Phi!\left(\Phi^{-1}§-\frac{|\delta|_2}{\sigma}\right),
]
若 (\mathbb{P}(X\in S)\le p)，则
[
\mathbb{P}(Y\in S)\le \Phi!\left(\Phi^{-1}§+\frac{|\delta|_2}{\sigma}\right),
]
其中 (\Phi) 是标准正态分布的分布函数。

因此对类别 (A)，有
[
\mathbb{P}(f(x+\delta+\varepsilon)=A)

\mathbb{P}(Y\in S_A)
\ge
\Phi!\left(\Phi^{-1}(p_A)-\frac{|\delta|_2}{\sigma}\right).
]

对任意 (c\neq A)，有
[
\mathbb{P}(f(x+\delta+\varepsilon)=c)

\mathbb{P}(Y\in S_c)
\le
\Phi!\left(\Phi^{-1}(p_B)+\frac{|\delta|_2}{\sigma}\right).
]

为了保证 (g(x+\delta)=A)，只需对所有 (c\neq A) 满足
[
\mathbb{P}(f(x+\delta+\varepsilon)=A)

\mathbb{P}(f(x+\delta+\varepsilon)=c).
]
由上面的上下界，一个充分条件是
[
\Phi!\left(\Phi^{-1}(p_A)-\frac{|\delta|_2}{\sigma}\right)

\Phi!\left(\Phi^{-1}(p_B)+\frac{|\delta|_2}{\sigma}\right).
]
由于 (\Phi) 单调递增，上式等价于
[
\Phi^{-1}(p_A)-\frac{|\delta|_2}{\sigma}

\Phi^{-1}(p_B)+\frac{|\delta|_2}{\sigma}.
]
整理得
[
\Phi^{-1}(p_A)-\Phi^{-1}(p_B)

2\frac{|\delta|_2}{\sigma}.
]
故当
[
|\delta|_2
<
\frac{\sigma}{2}\Big(\Phi^{-1}(p_A)-\Phi^{-1}(p_B)\Big)
]
时，必有
[
g(x+\delta)=A.
]

因此随机平滑的认证半径为
[
\boxed{
R=
\frac{\sigma}{2}\Big(\Phi^{-1}(p_A)-\Phi^{-1}(p_B)\Big)
}.
]

若进一步采用二分类式或 top-1 / 其余类合并的写法，即令 (p_B=1-p_A)，则
[
R

\frac{\sigma}{2}\Big(\Phi^{-1}(p_A)-\Phi^{-1}(1-p_A)\Big).
]
由 (\Phi^{-1}(1-p)=-\Phi^{-1}§)，化简为
[
\boxed{
R=\sigma\Phi^{-1}(p_A)
}.
]

实际中 (p_A) 往往由采样估计。若采样得到的是 (p_A) 的置信下界 (\underline p_A)，则对应的可认证半径写为
[
\boxed{

[NIPS2020] Certifying Confidence via Randomized Smoothing

传统随机平滑只关注 hard label 的多数表决, 但假如基分类器在最多数的类别上仅有微弱优势 (51% 是安全的), 依然是有问题的. 指出了基分类器的置信度问题可能导致虚假的高鲁棒性, 下面 EMNLP2025 Oral 那篇论文在大模型领域将这一点用置信度 α\alphaα 给概括了

[LD4C2022] Certified Robustness via Locally Biased Randomized Smoothing

LD4C (Learning for Dynamics and Control Conference) 在2019年举办第一届, 整体录用率在25%-30%之间, 偏向AI+物理, 有一些机器人与控制交叉领域的工作

TLDR:

随机平滑最初始的假设是加入零均值且数据无关的高斯噪声, 且需要做准确率与鲁棒性的 trade-off. 本篇论文设置输入图像的自适应噪声, 噪声分布有均值偏移, 均值来自输入图像在 Support Set 中的 KNN 结果, 偏置为数据点指向 K 近邻的多数类邻结点的质心. 结果是在相同的认证半径内, 准确率下降的可能性被降低了

输入图像与 Support Set 的 KNN: 一张图片 xxx 输入 →\rightarrow→ 通过高维索引查出最近的 KKK 个训练样本 →\rightarrow→ 根据邻居位置描绘出局部的几何安全边界 →\rightarrow→ 代入闭式解公式，瞬间算出最佳的平移向量 μ(x)\mu(x)μ(x) 和防御半径 RRR →\rightarrow→ 将加上了偏移量的点喂给冻结的神经网络，得出最终分类。

由 KNN 得到偏置函数: 由数据点指向 K 近邻的多数类邻结点的质心

μKNN(x)=1∣NK,c(x)∣∑xi∈NK,c(x)xi\mu_{KNN}(x) = \frac{1}{|\mathcal{N}_{K, c}(x)|} \sum_{x_i \in \mathcal{N}_{K, c}(x)} x_i μKNN​(x)=∣NK,c​(x)∣1​xi​∈NK,c​(x)∑​xi​

传统的随机平滑是“在悬崖边上闭着眼睛乱晃”（极易掉下去，准确率低）；而 KNN 偏置是“先把你拉回安全的安全区中心，然后再让你晃”。因此，在相同的认证半径内，准确率下降的可能性确实被降低了

优点:

  1. 在保证鲁棒性的同时尽量提升准确率
  2. 拥有闭式解, 不需要蒙特卡洛采样
  3. 传统高斯平滑无法改变线性分类器的决策边界, 本文可以
  4. 数学理论严密

缺点:

  1. KNN 的隐式计算成本不低
  2. KNN 在高维空间的维度灾难
  3. 需要用到大规模数据集
  4. 偏置计算的脆弱性

[NIPS2022] Invariance-Aware Randomized Smoothing Certificates

随机平滑是一种黑盒防御方法, 无法利用真实世界的物理规律, 算出的认证半径保守. 如果模型对某些变换是鲁棒的, 把先验写进数学公式里能得到更大的安全半径

[arXiv:2404.05350v1] Certified PEFTSmoothing: Parameter-Efficient Fine-Tuning with Randomized Smoothing

L2范数对抗攻击就是在寻找一个能让模型出错的恶意噪声 δ\deltaδ 时，严格限制这个噪声的总能量（即 L2 范数）不超过某个极小的阈值 ϵ\epsilonϵ。数学表达即为：∣∣δ∣∣2≤ϵ||\delta||_2 \leq \epsilon∣∣δ∣∣2​≤ϵ

随机平滑具体的运作步骤如下：

  1. 疯狂加噪：当输入一张图片（比如猫）时，我们不直接把它喂给模型，而是给这张图片加上成百上千次不同的高斯噪声，生成一大批“毛毛糙糙”的带噪图片。
  2. 批量预测：把这成百上千张带噪图片分别送进基础模型 fff 中，让它给出每一次的预测结果。
  3. 多数投票（Majority Vote）：统计所有的预测结果。如果这 1000 次预测里，有 800 次预测是“猫”，150 次是“狗”，50次是“汽车”，那么平滑模型 ggg 的最终输出就是得票数最高的“猫”。

黑盒模型通过零阶优化也能做类似的训练. 在图片周围加上一些可调整的像素, 通过调整像素来比较输出正确答案的概率(软标签), 反向估计梯度的近似值

Certified Robust 认证鲁棒性/可证明的鲁棒性: 随机平滑技术中, 算出安全半径 RRR 后, 只要攻击者添加的噪声能量不超过 RRR, 那么噪声不会影响预测结果

图像可以看作高维空间中的一个点，对抗扰动就是对这个点在空间中的位移。随机平滑的意义在于，它为这个点计算出了一个认证半径 RRR；只要黑客对图像的扰动位移不超过这个半径 RRR，平滑模型输出的最终预测结果就获得了绝对安全的数学保证，绝不改变

对于这篇论文. 训练数据有带噪图片, 那对新的带噪图片也能进行识别. 但是基础模型训练时少有特意的带噪训练, 传统方法是训练时加入带噪数据或图片输入前手动去噪. 作者把带噪图像的识别也识别为一种下游任务, PEFT 微调 Vision Encoder 来去除噪声的影响

但是论文存在严重的问题:

  1. PEFT+随机平滑没办法解决随机平滑刚需的多次加噪输入图片的问题, 推理时间长
  2. 防御类型单一: L2就只能防御L2

[ACL2020] Certified Robustness to Text Adversarial Attacks by Randomized Smoothing

将随机平滑引入文本领域, 将高斯噪声替换成同义词替换

  1. 构造同义词典
  2. 输入句子作随机同义词替换, 生成多个变体句子
  3. 目标分类器处理所有变体句子, 多数投票

结论: 最多替换 kkk 个但从. 多数投票结果不会改变. BERT 时代的工作

[NAACL2024] Advancing the Robustness of Large Language Models through Self-Denoised Smoothing

也是 [arXiv:2307.07171v1] Certified Robustness for Large Language Models with Self-Denoising

TLDR: 随机平滑在 LLM 上的应用, 难点在于图像是连续的, 直接加噪简单, 文本是离散的, 加噪难度大. 作者用文本的随机遮盖替换图像的加噪过程

让 LLM 做自己的去噪器, 对输入随机掩码, 让 LLM 补全缺失词, 再用恢复好的文本让 LLM 去做最终的任务预测, 多个问题作大多数投票. 与上面那篇论文其实是类似的, 只不过上面的论文是做图像扰动, 这篇论文做的是文本扰动.

是上面那篇论文在大模型时代的延伸.

[NIPS2023] RS-Del: Edit Distance Robustness Certificates for Sequence Classifiers via Randomized Deletion

将随机平滑引入到离散序列分类任务中, 可以解决替换, 删除, 插入等序列变长扰动. 不完全是 NLP 领域的工作, 更多是针对序列没有特意考虑文本模态, 针对二分类任务. 提出了随机删除平滑机制的基础理论框架

  1. 随机删除, 生成随机变体子序列
  2. 构建平滑分类器: 多数票决, 将子序列送入基础分类器预测, 利用 Clopper-Pearson 精确置信区间算出置信下界 pAp_ApA​ (使用 Clopper-Pearson 精确置信区间, 即有多少的把握认为胜率最低是多少)
  3. 求解认证编辑距离半径: 利用 Ney-Pearson 引理, 找到认证半径

基础分类器是 MalConv(1D-CNN) + 全局最大池化, 得到固定维度的向量, 再送入 MLP

[EMNLP2024 Findings] CERT-ED: Certifiably Robust Text Classification for Edit Distance

是上面那篇论文的延续, 作者基本也是同一批人. 将随机删除机制扩展到 NLP 文本领域, 将 RS-Del 扩展到多分类场景上, 在 token 级操作, 提出认证基数代替原本的汉明距离 (就是那个把一个字符串变成另一个字符串需要操作的最小次数)

文本处理中的数学认证是难以识别语义的, 比如 “我喜欢你” 与 “我喜欢她” 是两种完全不同的语义, 但是二者的汉明距离只有 1, 某种程度上这属于刻舟求剑. 编辑距离 (包括插入删除替换) 也是同理

认证基数

基于汉明距离的度量需要保证句子定长, 无法处理插入和删除操作. 传统方法算出的安全半径 RH=5R_H=5RH​=5 代表在保持句子定长时替换 5 个词可以保持结果不变. 但是 CERT-ED 算出的安全半径 RE=3R_E=3RE​=3 代表能防 3 步增删改, 无法比较这两种安全半径谁更安全.

认证基数: 对于给定的输入文本，在算出的安全半径内，攻击者到底能组合出多少种不同的人类可读句子（或 Token 序列）？这个总数量，就是认证基数. 认证基数将基于汉明距离和编辑距离得到的安全半径都转换为统一度量, 即文本变体的数量. 实际计算时是利用了莱文斯坦球的体积, 自动机+动态规划的算法

  • 汉明距离的基数增长（算数级）：如果允许 1 步替换，攻击者只能从原句中挑一个词，换成词典里的其他词。产生的变体数量大约是 3 万种。
  • 编辑距离的基数增长（指数级）：如果允许 1 步编辑（比如插入），攻击者不仅可以替换，还可以在句子开头、中间任意两个词之间、句子末尾等所有可能的缝隙中，插入词典中的任意一个词。产生的变体数量瞬间飙升到 几十万甚至上百万种。

[TMLR2025.05] SmoothLLM: Defending Large Language Models Against Jailbreaking Attacks

许多 jailbreak prompt 对字符级的扰动很脆弱, 于是作者对输入 prompt 作多个随机扰动, 再分别送进模型, 根据结果判断原 prompt 是否是攻击

[ICLR2025] Certified Robustness Under Bounded Levenshtein Distance

莱文斯坦距离 (Levenshtein Distance): 最开始在通信领域被提出, 是上面两篇论文编辑距离的更严谨的定义.

这篇文章与 CERT-ED 还是有区别的

  • CERT-ED: 在输入空间“稀释”攻击, 既然对抗者试图通过微小的增删改来寻找模型的决策边界，CERT-ED 干脆用随机删除的概率分布把输入“模糊化”
  • LipsLev: 限制模型的几何空间剧烈波动, 在莱文斯坦距离下限制卷积层的 Lipschitz 常数, 强制规定输入每发生一个字符的编辑变化，模型输出 logits 的变化幅度被严格锁定在一个常数上限 GGG 内

LipsLev: 针对莱文斯坦距离计算卷积分类器 Lipschitz 常数, 能训练出 1-Lipschitz 文本分类器, 仅适用单次前向传播. 这种方法还无法用到大语言模型上, 最多是在大模型前训练一个 CNN 文本分类器, 由这个分类器确认输入是否存在恶意攻击

Lipschitz 常数: 在数学上，如果一个函数 fff 满足 Lipschitz 连续条件，意味着存在一个常数 LLL，使得对于任意的输入 xxx 和 yyy，都满足：

∣∣f(x)−f(y)∣∣≤L⋅∣∣x−y∣∣||f(x) - f(y)|| \le L \cdot ||x - y|| ∣∣f(x)−f(y)∣∣≤L⋅∣∣x−y∣∣

用通俗的话来说，Lipschitz 常数 LLL 就是模型输出变化的“最高限速”。

虽然理论极其完美，但在工程实践中，Lipschitz 常数面临两个致命痛点：

计算是 NP-Hard 的：精确计算一个深层非线性神经网络（如 ResNet 或 Transformer）的全局 Lipschitz 常数在计算复杂性上被证明是极其困难的。我们通常只能计算出一个“上限（Upper Bound）”。

乘积效应导致的“维度灾难”：如果你计算每一层的常数然后相乘，一个 50 层的网络算出来的全局 LLL 可能会大到不可思议（比如 101510^{15}1015）。这种极其松散的边界在数学上毫无意义。而如果强行把每一层的 LLL 都限制为 1（比如通过正交权重初始化），又会严重限制神经网络的表达能力（Expressivity），导致模型在干净数据上的准确率大幅下降。

[EMNLP2019] Certified Robustness to Adversarial Word Substitutions

IBP (Interval Bound Propagation区间边界传播算法): 对抗攻击是在原始输入上加微小扰动, 扰动后的输入被限制在高维空间 x′∈[x−ϵ,x+ϵ]x' \in [x - \epsilon, x + \epsilon]x′∈[x−ϵ,x+ϵ] 内, IBP 的目标是逐层计算这个输入区间在通过神经网络后产生的激活值的绝对上下界

如果目标类别的最差情况（下界） 依然大于 任何其他错误类别的最好情况（上界），我们就从数学上绝对证明了：在这个 ϵ\epsilonϵ 半径内，没有任何对抗扰动能让模型分类错误。这就是所谓的“可认证鲁棒”。

  • 优点: 速度快. 前向传播即可
  • 缺点: ① 边界爆炸. 深度越大, 区间越宽, 下界可能会远低于实际最小可能值; ② 只能处理定长扰动, 如同义词替换, 不能处理插入或删除等改变输入序列长度的扰动

[ICLR2025] Certified Robustness Under Bounded Levenshtein Distance

Levenshtein Distance / Edit Distance: 衡量两个序列相似度的指标, 定义为将一个字符串转换为另一个字符串, 所需最少得单字符编辑次数(包括插入, 删除, 替换). 这种方法需要改变文本序列长度, 对深度学习模型失效

Lipschitz 连续性: 如果一个神经网络函数 fff 满足 Lipschitz 连续，且常数为 KKK，则对于任意输入 x1x_1x1​ 和 x2x_2x2​，其输出变化都有上限：
