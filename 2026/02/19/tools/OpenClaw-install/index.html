前置准备

一台云服务器

现在很多云服务厂商都提供了可以安装 OpenClaw 镜像的服务器, 也因此出了很多优惠.

需求: 在部署 OpenClaw 的同时能额外部署一个专门用于个人生活记录的 bot, 两个 bot 使用2核2G有点不够用, 所以我的目标就是2核4G. 不过后续在使用中我发现可以让 OpenClaw 同时完成后一个 bot 的功能, 所以实际上这个性能是有很大余量的.

购买服务器: 选用海外服务器能在访问国内网络的同时访问国外网络, 最优选是香港, 但是能否买到香港服务器需要看运气, 其他像新加坡和东京的节点也能接受, 反正又不是打fps游戏. 综合比较下来我使用的是腾讯云的轻量应用服务器, 新加坡节点2核4G30M带宽, 60GBSSD盘 1.5T 月流量, 另外1元选购了50GB的云盘, 可以用来备份或重装系统. 腾讯云服务器选购网址

AI应用服务商: 我看主流使用是 MiniMax 或 Kimi, 我选用的是 MinimaxMax 最基础的 Coding Plan, 一个月 29r, 40prompts/5h, 这里一个 prompt 一般可以完成十几次回复, 对于我的个人使用是绰绰有余的.

QQ开发平台: 接入QQ机器人需要使用, 这个与使用一个用户作为机器人不同, 这是官网提供的机器人账号, 我感觉操作起来会更简单些, 但同样限制也会相对多一些

安装

云服务器的安装参考云上OpenClaw快速接入QQ指南, 如果有更新的模型需要在腾讯云的服务器上自定义json来接入
