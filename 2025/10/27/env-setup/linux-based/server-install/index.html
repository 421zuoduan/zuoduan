Mac 烧录 Ubuntu 24.04 启动 U 盘

Created: 2025-10-27
Edited: 2026-03-25

myself

 596 words, 2 min 环境配置
server

推荐 ubuntu22.04

方式一：使用 balenaEtcher（推荐）

准备工作

  • 一个 至少 8GB 的 U 盘
  • 已下载好的 Ubuntu 24.04 ISO 文件
  • 注意：烧录会清空 U 盘中的所有内容
  • ISO 文件应放在 Mac 本地磁盘 中，不要放在目标 U 盘里

操作步骤

  1. 下载并安装 balenaEtcher, RM64 适用于 M 系列 mac
  2. 插入 U 盘
  3. 打开 balenaEtcher
  4. 点击 Flash from file
  5. 选择本地的 ubuntu-22.04...iso 文件
  6. 点击 Select target
  7. 选择目标 U 盘
  8. 点击 Flash
  9. 等待烧录完成

说明

  • 烧录完成后，macOS 可能会弹出：

    The disk you inserted was not readable by this computer.

    这是正常现象，直接点击 Ignore 即可。

优点

  • 图形界面操作简单
  • 不容易选错磁盘
  • 适合大多数用户

方式二：使用命令行烧录 U 盘

准备工作

  • 一个 至少 8GB 的 U 盘
  • 已下载好的 Ubuntu 22.04 ISO 文件
  • 注意：烧录会清空 U 盘中的所有内容
  • ISO 文件应放在 Mac 本地磁盘 中，不要放在目标 U 盘里
  • 务必确认 U 盘对应的磁盘编号，否则可能误写入其他磁盘

操作步骤

  1. 插入 U 盘
  2. 打开终端
  3. 查看当前磁盘列表：
  1. 找到你的 U 盘对应的设备编号，例如 /dev/disk4
  2. 卸载整个 U 盘（将 disk4 替换为你的实际编号）：
  1. 使用 dd 将 ISO 写入 U 盘（将 ISO 路径和磁盘编号改成你自己的）：
  1. 等待写入完成
  2. 写入完成后执行同步并弹出 U 盘：

参数说明

  • if=：输入文件，即 ISO 文件路径
  • of=：输出目标，即 U 盘设备
  • bs=4m：每次写入 4MB，通常更快
  • rdisk4 相比 disk4 通常写入速度更快

说明

  • 烧录过程中终端可能长时间没有输出，这是正常现象
  • 可以按 Ctrl + T 查看 dd 当前进度

  • 烧录完成后，macOS 可能会弹出：

    The disk you inserted was not readable by this computer.

    这是正常现象，直接点击 Ignore 即可。

优点

  • 不需要额外安装软件
  • 适合熟悉终端的用户
  • 写入过程直接、稳定

风险

  • 如果磁盘编号写错，可能会误覆盖其他磁盘的数据
  • 执行 dd 前建议再次用 diskutil list 确认目标设备
