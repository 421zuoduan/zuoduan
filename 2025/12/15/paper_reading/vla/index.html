VLA 基本架构

VLA 的 action token 可以对标 VLM 里生成的 language token, 当前 VLA 基座模型分为自回归范式, 自回归+diffusion头范式

自回归范式

以 OpenVLA 为例, OpenVLA 将 llama 词表中用到最少的 256 个词替换为 action token, 模型预测时将只在这 256 个 词上选取. 机器人执行动作时, 动作会被拆解为 NNN 个维度, 每个维度是一个连续的标量, 这个标量被拆解为 256 个区间, 标量上下界是训练数据中的上下界. 一般会有 Δx, Δy, Δz, Δroll, Δpitch, Δyaw, gripper 七个要预测的 token, 如 RT-2 等模型还有 terminate 等额外 token. 单次生成 at=Rda_t = \mathcal{R}^dat​=Rd, 预测了单个时间步的动作

  • Δx, Δy, Δz: 末端执行器在三维空间里位置的增量
  • Δroll, Δpitch, Δyaw: 末端执行器朝向的增量
  • gripper: 夹爪的开合控制, 0 或 1

自回归+diffusion头范式

以 π0\pi_\mathrm{0}π0​ 为例, VLM 得到 last hidden state 后同时进行 HHH 个 action token 的同时预测, 经 diffusion 得到 At=RH×dA_t=\mathcal{R}^{H×d}At​=RH×d, 预测了 HHH 个时间步的动作

仿真环境

预测得到 token 后, 将 token 对应的动作在仿真环境中进行仿真得到结果, 比如用 robosuite 仿真环境跑 LIBERO

Efficient

[NIPS2025] EfficientVLA: Training-Free Acceleration and Compression for Vision-Language-Action Models

当前基于 diffusion 的 VLA 模型加速方法忽略了模型各个组件间的加速瓶颈, 作者研究表明这种范式的模型存在内存瓶颈和计算瓶颈 (GPU瓶颈和CPU瓶颈), 单一的 vision token 剪枝不能缓解计算瓶颈带来的问题. 基于此, 作者提出衡量瓶颈是在 LLM 还是在 diffusion 头上, 并做相应加速.

(1) 基于层间冗余性分析，对语言模块中功能无关紧要的层进行剪枝；(2) 通过任务感知策略优化视觉处理通路，选择紧凑且多样化的视觉tokens集合，在任务关键性与信息覆盖度之间实现平衡；(3) 采用策略性缓存和复用关键中间特征的方法，减轻基于迭代扩散的动作头内部的时间计算冗余。

通过相似度衍生的重要性指标定位语言模块的主要内存瓶颈
